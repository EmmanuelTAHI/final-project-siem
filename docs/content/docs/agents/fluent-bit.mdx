---
title: Fluent Bit (générique / conteneurs)
description: Envoyer des logs de conteneurs ou de fichiers arbitraires vers Log+
---

[Fluent Bit](https://fluentbit.io/) est un collecteur de logs open source
très léger, adapté aux conteneurs (Docker, Kubernetes) comme aux serveurs
classiques.

## 1. Installer Fluent Bit

```bash
# Debian / Ubuntu
curl https://raw.githubusercontent.com/fluent/fluent-bit/master/install.sh | sh
```

Ou utilisez l'image officielle `fluent/fluent-bit` si vous êtes déjà en
conteneurs.

## 2. Configurer `fluent-bit.conf`

```ini
[SERVICE]
    Flush        5
    Log_Level    info

[INPUT]
    Name         tail
    Path         /var/log/*.log
    Tag          logplus.*

[OUTPUT]
    Name         http
    Match        logplus.*
    Host         votre-instance.example
    Port         443
    URI          /api/ingest/agent/logs/
    tls          on
    Format       msgpack
    Header       Authorization Bearer logplus_agt_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    Header       Content-Type text/plain
    Body_Key     message
```

<Callout type="info">
  Log+ attend une ligne de texte par événement (`text/plain`). Si votre
  source produit du JSON structuré, utilisez un filtre `lua` ou `record_modifier`
  en amont pour sérialiser chaque enregistrement en une seule ligne avant l'output.
</Callout>

## 3. Kubernetes (DaemonSet)

Pour un cluster Kubernetes, déployez Fluent Bit en DaemonSet avec la même
configuration `[OUTPUT]`, en pointant `[INPUT]` sur
`/var/log/containers/*.log` — voir la documentation officielle Fluent Bit
pour le manifeste complet.

## 4. Vérifier

```bash
sudo systemctl restart fluent-bit
sudo journalctl -u fluent-bit -f
```

Vérifiez ensuite l'apparition des logs dans **Logs** sur le tableau de bord
Log+.
